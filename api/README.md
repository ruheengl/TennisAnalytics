# API package

`api/` serves normalized data products and ML-oriented endpoints from the
SQLite artifact generated by `pipeline/features.py` (`data/features/player_features.sqlite`).

## Endpoints

- `POST /cluster`
  - Runs KMeans-style clustering over selected numeric attributes.
  - Accepts algorithm params: `k`, `distance_metric` (`euclidean|manhattan|cosine`), `scaling` (`none|zscore|minmax`), `max_iter`, `seed`.
  - Supports optional pre-filtering via `filters` payload.
  - Caches results keyed by normalized payload hash (`cluster_request_id`).
  - Returns D3-focused metadata: centroid stats, cluster ids, sampled player labels, confidence + quality metrics.

- `GET /clusters/{cluster_request_id}/players`
  - Returns paginated players assigned to cached cluster results.
  - Filters: `cluster_id`, `min_confidence`.
  - Includes nearest-neighbor support via `similar_to=<player_id>&similar_limit=<n>` for similar play-style lookups.

- `GET /players/search?q=...`
  - Text search over `player_id`.
  - Optional constraints by `cluster_request_id` and `cluster_id`.

- `POST /players/query`
  - Attribute threshold filtering with operators `gt|gte|lt|lte|eq|between`.
  - Supports optional cluster constraint (`cluster_request_id`, `cluster_label`).

- `GET /players/{player_id}/metrics/timeseries`
  - Retrieves player metric time series for `elo`, `ace_pct`, `break_points_won_pct`, `win_pct`.
  - Supports optional date bounds, sampling limit, smoothing (`none|ema|moving_average`) and regression (`ols|theil_sen`) controls.
  - Returns both raw/smoothed points and derived trend annotations (slope, confidence, CI, change points).

- `GET /players/{player_id}/metrics/degradation`
  - Runs configurable degradation checks over trend output.
  - Criteria include negative slope threshold, drawdown threshold, and sustained decline windows.
  - Returns trend annotations plus degradation flags and matched decline windows.

- `POST /predict`
  - Loads `data/models/match_outcome_tree.pkl` generated by `pipeline/modeling.py`.
  - Predicts win probability from a feature payload keyed by training feature names.
  - Returns an explanation payload for visualization:
    - `top_contributing_features` (ranked by split-margin contribution)
    - `path_summary` (leaf details and traversed split rules)
  - Includes `used_default_medians_for` listing missing features imputed from training medians.

## Run

```bash
uvicorn api.server:app --reload
```

## Notes

- The service expects `data/features/player_features.sqlite` to already exist.
- Clustering cache is in-memory and resets when process restarts.
